## デザイン方針

### テーマ: Dedalus/Linear寄りのダークテーマ

白背景は断念。参考サイト3つのうち2つがダーク。エンタープライズ×セキュリティのプロダクトにはダークテーマが信頼感を与える。

### フォント

| 用途   | フォント               | 理由                             |
| ------ | ---------------------- | -------------------------------- |
| 見出し | Inter (weight 600-700) | SaaS業界標準。Merge/Vercelが使用 |
| 本文   | Inter (weight 400)     | 読みやすさ                       |
| コード | JetBrains Mono         | 開発者向け、Dedalus風            |

**Syneは使わない**。Syneの幾何学的な字形がテンプレート感の原因。

### セクション構成

1. Header: ダーク、透過blur
2. Hero: 左テキスト + 右にライブ風モニターUI
3. Social Proof: ロゴバー（信頼企業ロゴ）→ v2で追加
4. Problem: 3リスクカード
5. Solution: Split-Planeアーキテクチャ図
6. Features: 左右交互レイアウト + インタラクティブUI
7. Trust: ユースケース
8. Pricing: 月額/年額トグル
9. CTA: グラデーション背景
10. Footer

### アニメーション方針

- Framer Motionは維持
- **CSSカスタムプロパティではなく、標準Tailwindクラスのみ使用**（カスタムトークンのバグを排除）
- Hero: 左テキストstaggerフェードイン + 右UIはタイプライター風ログ表示
- セクション: whileInViewでフェードイン（AnimateIn共通コンポーネント）
- カード: hover時のborder-color変化 + subtle translateY

### 技術制約

- **`@theme`ブロックのカスタムカラートークンは使わない** → 標準のTailwindカラー（zinc, violet等）を直接使用
- globals.cssはミニマルに（スムーズスクロール + アニメーションkeyframesのみ）
- フォントはnext/font/googleのInterとJetBrains Monoに変更
