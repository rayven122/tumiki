# Tumiki エンドユーザーポータル 画面設計書

**対象ユーザー:** 一般社員（AIエージェント利用者）
**スコープ:** セルフサービスポータル（ツール確認・操作履歴・権限申請）
**アーキテクチャ位置:** Control Plane（SaaS）側のUI。Data Planeのログはメタデータのみ参照。
**認証:** Entra ID / Keycloak SSO（企業ID基盤連携）

---

## 画面一覧（サイトマップ）

```
Tumiki エンドユーザーポータル
│
├── 1. ログイン
│
├── 2. ダッシュボード（ホーム）
│
├── 3. マイツール
│   ├── 3-1. ツール一覧
│   ├── 3-2. ツール詳細
│   └── 3-3. ツールカタログ（未承認ツールの閲覧・申請）
│
├── 4. 操作履歴
│   ├── 4-1. 履歴一覧
│   └── 4-2. 履歴詳細
│
├── 5. 権限申請
│   ├── 5-1. 申請一覧（自分の申請）
│   ├── 5-2. 新規申請フォーム
│   └── 5-3. 申請詳細・ステータス
│
└── 6. 設定
    ├── 6-1. プロフィール
    └── 6-2. 通知設定
```

- ペイロード（プロンプト内容・レスポンス）は一切表示しない（Data Plane完結の原則）

## 技術的な制約・設計原則

1. **Data Plane完結の原則**: ポータルに表示するのはControl Planeが保持するメタデータのみ。MCPペイロード（プロンプト・レスポンス）は一切表示しない。

2. **表示可能なデータ**:
   - ツールメタデータ（名前、説明、操作一覧）
   - ポリシー設定（ロール、権限）
   - 匿名化統計（リクエスト数、エラー率）
   - タイムスタンプ、ステータス、リクエストID

3. **表示不可のデータ**:
   - MCPペイロード（プロンプト内容、レスポンスデータ）
   - APIキー・認証トークン
   - PII（個人情報）
   - 他ユーザーの操作履歴

4. **認証**: SSO（Entra ID / Keycloak）のみ。ID/PW直接入力は不可。

5. **レスポンシブ対応**: デスクトップ優先だが、タブレットでの利用も想定。

6. **アクセシビリティ**: WCAG 2.1 AA準拠を目標。
