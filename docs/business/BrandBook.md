# Tumiki ブランドブック

## 1. ブランドステートメント

**Tumikiは、企業が生成AIと外部ツールを安全・自由につなぐための、クラウド統制基盤です。**

積み木のように構成可能で、柔軟かつ堅牢な統制環境を通じて、AI導入を“安心して使えるもの”に変えていきます。私たちは、企業AIの“アクセルとブレーキ”を同時に提供します。

---

## 2. 成り立ち・ネーミングの意味

「**Tumiki（積み木）**」という名前には、一見すると子どもっぽく感じられる面もありますが、**B2B SaaS・セキュリティ・AI統制基盤**としても十分に**深い意味づけ・再解釈**が可能です。以下の観点から、Tumikiという名前を“尖らせて再定義”することができます。

### 🧱 Tumiki（積み木）の本質的な意味

#### ① **構成可能性（Composable Architecture）**

> 積み木は一つ一つは単純でも、**自由自在に組み合わせて高度な構造をつくれる**。

* AI × MCP × API という「機能ブロック」を企業ごとに組み立てられるTumikiの特性と一致。
* → \*\*ノーコードでつなぎ、ルールで統制し、柔軟に再構成できる“構成可能セキュリティ”\*\*という世界観。

#### ② **安全な成長の象徴（Controlled Exploration）**

> 積み木は“遊び”ながらも、安全に創造性を育む教育道具。

* Tumikiも、AIエージェントという強力な技術を\*\*“安全に業務に取り入れるための土台”\*\*を提供。
* → 「**安全に試し、守りながら育てる**」というメッセージを内包。

#### ③ **順序性・積み重ね（Stacking Layers）**

> 積み木は、下から順に積まないと崩れる。「**秩序ある構造**」が必要。

* Tumikiでは「アクセス制御→接続設定→AI実行→ログ監査」という階層構造があり、
* → **企業システムにAIを“順序よく、安全に”組み込むためのレイヤー設計**という意味と接続。

### 💡 言語的再解釈：「Tumiki」は略語にもなりうる

#### 例：T.U.M.I.K.I.（ブランディング補強案）

| 項目    | 意味                      |
| ----- | ----------------------- |
| **T** | Trust（信頼）               |
| **U** | Usability（使いやすさ）        |
| **M** | Modularity（モジュール性）      |
| **I** | Integration（統合性）        |
| **K** | Knowledge control（知識統制） |
| **I** | Intelligence（AIの利活用）    |

→ プレゼンやピッチでは「**Tumikiは信頼・制御・拡張性を積み上げる構成基盤**です」と補足可能。

### 🏗️ ブランドストーリーへの応用（ストーリーテリング）

> **「Tumiki」は、生成AIという“巨大な知能”を、積み木のようにコントロールし、
> 安全に積み重ねていくための“統制と創造のレイヤー”です。」**

* 子どもが遊ぶ積み木のように、誰でもドラッグ＆ドロップでAI接続を作れる。
* ただしその積み木は、「崩れないように」「中身を記録しながら」「勝手に暴走しないように」制御されている。
* → **自由と統制の両立＝Tumikiが目指す世界観**。

### ✅ 結論：Tumikiという名前を使い続けるために

* 「積み木＝構成・制御・安全な成長」の意味を前面に出す
* ロゴやタグラインでメッセージ補強する：

  > 例：**「Tumiki – 組み上げろ、AIの安全な力を」**
* ストーリーの語り方次第で、**可愛いが“信頼される”ブランド名に変わる**
