# 追加開発特化パターン（同フォーマット）

## パターン3: モバイルアプリ開発チーム

**iOS・Android・クロスプラットフォーム開発**

### Tumiki MCP Manager設定

- **GitHub MCP**（ソースコード管理）
- **App Store Connect MCP**（iOS配布）
- **Google Play Console MCP**（Android配布）
- **Firebase MCP**（バックエンド・分析）
- **TestFlight MCP**（ベータテスト）

### 各AIエージェントの専門化

- **iOSエージェント**
  → Swift/SwiftUI + App Store Connectアクセス
- **Androidエージェント**
  → Kotlin/Jetpack Compose + Google Playアクセス
- **クロスプラットフォームエージェント**
  → React Native/Flutter + 両OS対応
- **バックエンドエージェント**
  → Firebase + Push通知アクセス
- **QA・リリースエージェント**
  → TestFlight + 審査対応アクセス

### 実現される協働

- 各プラットフォーム並行開発→統一バックエンド連携
- 自動ビルド→TestFlight/内部テスト配布
- ストア審査→自動対応・リジェクト修正
- アプリ分析→Firebase Analytics自動レポート

### 協働フロー例：

**企画・設計 → iOS/Android並行開発 → 統合テスト → ストア申請 → リリース・分析**
