## 1. 概要

### 1.1 目的

Tumiki for Desktop は、ユーザーのローカル環境でMCPサーバーを実行し、Tumiki Cloudと連携するデスクトップアプリケーション。

### 1.2 解決する課題

| 課題 | 解決方法 |
|------|---------|
| ステートフルMCPサーバーの対応 | ローカルでOAuthトークン・セッションを永続化 |
| ローカルファイルアクセス | ユーザー環境で直接ファイル操作 |
| 機密データの保護 | クラウドに送信せずローカルに保持 |
| Cloud Runの制限 | ローカル実行で制限を回避 |

## 3. 技術スタック

### 3.1 フレームワーク選定

| 候補 | メリット | デメリット | 採用 |
|------|---------|-----------|------|
| **Electron** | エコシステム充実、実績多数 | バンドルサイズ大 | ◎ |
| Tauri | 軽量、Rust製 | エコシステム小、学習コスト | △ |
| Neutralinojs | 超軽量 | 機能制限 | × |

**採用: Electron**

理由:
- Node.jsとの親和性（既存tumikiコードベースと統一）
- MCPサーバー（npm パッケージ）の実行が容易
- 豊富なプラグイン・ライブラリ

### 3.2 技術構成

```yaml
# 技術スタック

framework:
  runtime: Electron 34+
  node: 22+

frontend:
  framework: React 19
  language: TypeScript 5.x
  styling: Tailwind CSS
  state: Jotai
  ui_components: Radix UI (packages/ui と共通)

backend:
  language: TypeScript
  ipc: Electron IPC
  database: better-sqlite3 (オプション)

build:
  bundler: electron-vite
  packager: electron-builder

testing:
  unit: Vitest
  e2e: Playwright

ci_cd:
  build: GitHub Actions
  release: electron-builder (auto-update)
  signing:
    macos: Apple Developer ID
    windows: EV Code Signing Certificate
```

### 4.2 ユースケース

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                     UC-1: MCPサーバーの追加                                  │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  アクター: ユーザー                                                          │
│                                                                             │
│  事前条件:                                                                   │
│  - Tumiki for Desktop がインストール済み                                     │
│  - Tumiki Cloud に接続済み                                                  │
```

## 5. 通信プロトコル

### 5.1 WebSocketメッセージ仕様

```typescript
// 基本メッセージ構造
type BaseMessage = {
  type: MessageType;
  id: string;          // リクエストID（UUID v4）
  timestamp: number;   // Unix timestamp (ms)
};

type MessageType =
  | "mcp_request"      // Cloud → Desktop: MCPリクエスト
  | "mcp_response"     // Desktop → Cloud: MCPレスポンス
  | "status_update"    // Desktop → Cloud: 状態更新
  | "control"          // Cloud → Desktop: 制御コマンド
  | "ping"             // 双方向: ヘルスチェック
  | "pong"             // 双方向: ヘルスチェック応答
  | "error";           // 双方向: エラー

// MCPリクエスト (Cloud → Desktop)
type McpRequestMessage = BaseMessage & {
  type: "mcp_request";
  payload: {
    server: string;              // MCPサーバー名
    method: string;              // JSON-RPC method
    params?: Record<string, unknown>;
  };
};

// MCPレスポンス (Desktop → Cloud)
type McpResponseMessage = BaseMessage & {
  type: "mcp_response";
  payload: {
    result?: unknown;            // 成功時
    error?: {                    // エラー時
      code: number;
      message: string;
      data?: unknown;
    };
  };
};

// 状態更新 (Desktop → Cloud)
type StatusUpdateMessage = BaseMessage & {
  type: "status_update";
  payload: {
    status: "connected" | "disconnected";
    servers: Array<{
      name: string;
      status: "running" | "stopped" | "error";
      tools?: string[];          // 利用可能なツール一覧
      error?: string;
    }>;
    version: string;             // アプリバージョン
    platform: "darwin" | "win32" | "linux";
  };
};

// 制御コマンド (Cloud → Desktop)
type ControlMessage = BaseMessage & {
  type: "control";
  payload: {
    action: "start_server" | "stop_server" | "restart_server" | "start_oauth";
    server: string;
    config?: Record<string, unknown>;
  };
};

// エラー
type ErrorMessage = BaseMessage & {
  type: "error";
  payload: {
    code: string;
    message: string;
    requestId?: string;          // 元のリクエストID
  };
};
```

```typescript
// apps/mcp-proxy/src/features/desktopGateway/connectionManager.ts

type DesktopConnection = {
  userId: string;
  organizationId: string;
  socket: WebSocket;
  servers: Map<string, ServerStatus>;
  connectedAt: Date;
  lastPingAt: Date;
};

class DesktopConnectionManager {
  private connections: Map<string, DesktopConnection> = new Map();

  // ユーザーIDで接続を取得
  getConnection(userId: string): DesktopConnection | undefined {
    return this.connections.get(userId);
  }

  // MCPリクエストを転送
  async forwardMcpRequest(
    userId: string,
    server: string,
    method: string,
    params: unknown
  ): Promise<unknown> {
    const connection = this.getConnection(userId);
    if (!connection) {
      throw new Error("Desktop not connected");
    }

    const serverStatus = connection.servers.get(server);
    if (!serverStatus || serverStatus.status !== "running") {
      throw new Error(`Server ${server} not available`);
    }

    // リクエスト送信・レスポンス待機
    return this.sendAndWait(connection, {
      type: "mcp_request",
      id: crypto.randomUUID(),
      timestamp: Date.now(),
      payload: { server, method, params },
    });
  }
}
```

```prisma
  /// ユーザーID

  user           User     @relation(fields: [userId], references: [id], onDelete: Cascade)

  /// 組織ID
  organizationId String
  organization   Organization @relation(fields: [organizationId], references: [id], onDelete: Cascade)

  /// デバイス情報

  platform       String               // "darwin", "win32", "linux"
  appVersion     String               // "1.0.0"

  /// 接続状態

  lastDisconnectedAt DateTime?

  /// 利用可能なMCPサーバー
  availableServers Json?              // ["filesystem", "gdrive", "slack"]

  createdAt      DateTime @default(now())
  updatedAt      DateTime @updatedAt
```

## 付録A: 技術的決定事項

### A.1 Electron vs Tauri

| 観点 | Electron | Tauri |
|------|---------|-------|
| バンドルサイズ | ~150MB | ~10MB |
| メモリ使用量 | 高 | 低 |
| Node.js互換性 | 完全 | 限定的 |
| MCPサーバー実行 | 容易 | 追加作業必要 |
| 学習コスト | 低 | 高 |
| エコシステム | 充実 | 発展中 |

**結論**: MCPサーバー（npmパッケージ）の実行が主目的のため、Node.js互換性が高いElectronを採用。

### A.2 状態管理

| 観点 | SQLite | JSON Files |
|------|--------|------------|
| クエリ性能 | 高 | 低 |
| 実装複雑度 | 中 | 低 |
| データ整合性 | 高 | 中 |
| 可読性 | 低 | 高 |

**結論**: 初期はJSON Filesで実装し、必要に応じてSQLiteに移行。
