# Pull Request 作成時のルール


以下のテンプレートに則って、PRを作成してください。
また、改行コードがそのまま表示されないように、以下を参考にして作成してください
```sh
echo "## 変更内容\n\n- サーバー一覧表示機能の実装\n- サーバー追加機能の実装\n- UIコンポーネントの追加（Alert Dialog, Badge）\n\n## 主な変更点\n\n1. サーバー一覧ページ（/servers）\n   - サーバーカードによる一覧表示\n   - 各サーバーのステータス表示\n\n2. サーバー追加機能\n   - サービス選択画面\n   - トークン入力画面\n\n3. UIコンポーネント\n   - Alert Dialog: 確認ダイアログ用\n   - Badge: ステータス表示用" | gh pr edit 2 --body-file -
```

```md
## 変更内容の概要
<!-- このPRで実装した変更内容を簡潔に説明してください -->

## 変更理由
<!-- なぜこの変更が必要なのかを説明してください -->

## 実装の詳細
<!-- 具体的な実装内容や変更点を説明してください -->

## テスト内容
<!-- どのようなテストを行ったか、または必要なテストを記載してください -->

## スクリーンショット（UI変更がある場合）
<!-- UIの変更がある場合は、変更前後のスクリーンショットを添付してください -->

## チェックリスト
- [ ] コードの変更が適切にテストされている
- [ ] 必要なドキュメントが更新されている
- [ ] コードスタイルガイドラインに従っている
- [ ] 不要なコメントアウトやデバッグコードが削除されている
- [ ] コミットメッセージが適切に書かれている

## 関連するIssue
<!-- 関連するIssue番号を記載してください -->
Closes #

## その他の注意事項
<!-- レビュアーに伝えておきたい情報があれば記載してください --> 
```
