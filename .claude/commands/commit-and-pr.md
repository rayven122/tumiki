現在ステージされているgitの変更を分析し、適切なコミットメッセージを生成してコミット、プッシュ、PR作成まで自動実行してください。

## 実行手順

1. **ブランチ確認と新規作成**
   - 現在のブランチを確認
   - **mainブランチの場合は必ず新しいブランチを作成してから処理を続行**
   - 新規ブランチ名は `feature/` + タイムスタンプまたは変更内容から自動生成

2. **Git状態の確認と分析**
   - 現在のブランチとステージされた変更を確認
   - 変更されたファイルの種類を分析してコミットタイプを決定

3. **コミットメッセージの自動生成ルール**
   - 新規ファイルのみ → `feat:`
   - 既存ファイル修正のみ → `fix:`
   - ファイル削除あり → `refactor:`
   - ドキュメントファイル(.md) → `docs:`
   - テストファイル → `test:`
   - 設定ファイル(package.json等) → `chore:`

4. **コミット実行（変更がある場合のみ）**
   - ステージされた変更がある場合のみコミット実行
   - 以下の形式でコミットメッセージを作成：

   ```
   [type]: [変更の概要]

   🤖 Generated with [Claude Code](https://claude.ai/code)

   Co-Authored-By: Claude <noreply@anthropic.com>
   ```

5. **プッシュとPR作成**
   - **コミットの有無に関わらず、ブランチをリモートにプッシュ**
   - GitHub CLIを使用してPR作成（`gh pr create --assignee @me` でアサインを設定）
   - **PR概要（タイトル・本文）は必ず日本語で記述する**
   - PR本文には変更概要、ファイル一覧、テストプランを含める

## コンテキスト取得

現在のgit状態とブランチ情報を取得:
!`git status`
!`git diff --cached --name-status`
!`git branch --show-current`
!`git log --oneline -3`

## 前提条件

- GitHub CLI (gh) がインストール・認証済みであること
- リモートリポジトリが設定されていること
- （変更がない場合でも実行可能）

## エラーハンドリング

- mainブランチでの直接コミットを防止（新規ブランチ作成を強制）
- GitHub CLI未インストールの場合は手動PR作成のURL表示
- プッシュ失敗時は上流ブランチ設定を試行
- 変更がない場合でもブランチプッシュとPR作成は実行
